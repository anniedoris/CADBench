# Welcome to CADBench!
This repo corresponds with our paper, CADBench: A Multimodal Benchmark for AI-Assisted CAD Program Generation. Here are some useful resources:

- [CADBench Paper](https://arxiv.org/abs/2605.10873)
- [CADBench Leaderboard and Project Page](https://anniedoris.github.io/CADBench/)
- [CADBench HuggingFace Dataset](https://huggingface.co/datasets/DeCoDELab/CADBench)

# Setup
1. First, fork the repo on Github and clone it locally. (Forking is needed to submit to the leaderboard via PR later on, but just cloning is sufficient for using the benchmark.)

2. Configure the conda environment:

```
cd CADBenchEval
mamba create --name CADBench python==3.11 cadquery pythonocc-core uv
conda activate CADBench
uv pip install -r requirements.txt && uv pip install numpy==2.1
```

# Testing Your Model on CADBench

### 1. Download Relevant Benchmark Modalities
To test your model on CADBench, you will first need to download the relevant benchmark modalities. For a mesh-to-CAD model, run:

```
python scripts/download_from_hf.py --model_type mesh
```

For an image-to-CAD model, run:

```
python scripts/download_from_hf.py --model_type image
```

This will save the relevant benchmark modalities under `benchmark_data/`. For the mesh-to-CAD models, you should see `benchmark_data/mesh/{split_name}` and `benchmark_data/noisy_mesh/{split_name}`. For the image-to-CAD models, you should see `benchmark_data/singleview/{split_name}`, `benchmark_data/multiview/{split_name}`, and `benchmark_data/pbr/{split_name}`.

### 2. Test Your Model
Using the modalities, test your model however you would normally provide either mesh or image inputs. Save your model's output using the following structure under the `tested_models/` directory. For an example of saving a mesh-to-CAD model, see `tested_models/cadrecode`. For an example of saving an image-to-CAD model, see `tested_models/gemini3.1`.

```text
{model_or_method_name}/
├── {modality_name}/
│   ├── r1/
│   │   ├── bench0.jsonl
│   │   ├── bench0F.jsonl
│   │   └── ...
│   ├── r2/
│   │   ├── bench0.jsonl
│   │   ├── bench0F.jsonl
│   │   └── ...
│   └── r3/
│       ├── bench0.jsonl
│       ├── bench0F.jsonl
│       └── ...
└── ...
```

Each jsonl file should correspond with a benchmark subset. Each row in a jsonl should correspond with one CAD sample and contain the "file_id" and "generated" fields, where "file_id" is the filename stem of the CAD sample (e.g. "00708740") and "generated" is the CAD program generated by your method:

```json
{"file_id": "00708740", "generated": "import cadquery as cq\nw0=cq.Workplane('ZX',origin=(0,76,0))\nr=w0.sketch().arc((-27,-87),(0,-100),(27,-88)).segment((26,-88)).segment((26,87)).arc((0,100),(-27,88)).segment((-26,88)).segment((-26,-77)).segment((-16,-77)).segment((-16,-87)).close().assemble().finalize().extrude(-152)"}
{"file_id": "00714815", "generated": "import cadquery as cq\nw0=cq.Workplane('YZ',origin=(-2,0,0))\nr=w0.workplane(offset=-8/2).cylinder(8,57).union(w0.sketch().rect(34,200).push([(-8,-79)]).circle(5,mode='s').finalize().extrude(12))"}
```

Note that currently we only support "generated" as CadQuery Python code, but our later eval pipeline could be adapted to other CAD programming languages. `r1`, `r2`, and `r3` correspond with different runs of the model -- if your method is deterministic, you can just save one run but keep it under the `r1` folder.

### 3. Run Metric Computation

To compute the metrics on your model's outputs, run the following:

```
conda activate CADBench && python scripts/run_eval_pipeline.py --model_dir tested_models/{model_or_method_name} --variable-name {solid_name} --num-workers {num_workers}
```

where `solid_name` is the final variable name assigned to the CAD solid in the model's generated code.

### 4. Submit Your Scores to the Leaderboard
You can submit your scores to the leaderboard by submitting a PR. To do this, do the following:

```
git checkout -b leaderboard/{model_or_method_name}
git add tested_models/{model_or_method_name}
git commit -m "Add leaderboard submission: {model_or_method_name}"
git push origin leaderboard/{model_or_method_name}
```

On your Github fork, create a pull request from the `leaderboard/{model_or_method_name}` branch to the base: `anniedoris/CADBench:main`. Please fill out the PR template accordingly!

## Generate Benchmark Subsets
The following scripts allow you to pull the subsets of the benchmark in the same way that we did.

### Bench 0: DeepCAD
To get bench 0, first download the DeepCAD vector format dataset from [this link](https://drive.google.com/file/d/1q2FW_rH89LnSlc2EZw_sFaZIh83e6Jzb/view?usp=drive_link). Unzip the files - they should end up in data/cad_vec. Also download the train/test split json from [this link](https://github.com/ferdous-alam/GenCAD/blob/main/data/filtered_data.json) and store it in the data/ folder.

Deduplicate the h5 files in the dataset using the following command:

```
python scripts/deduplicate_h5.py --input_data_dir data/cad_vec --output_data_dir data/cad_vec_dedup_test --split_json data/filtered_data.json
```

This should leave you with 6870 h5 vector files in data/cad_vec_dedup_test.

Next, generate the step files for each of the h5 files using the deepcad repo. Clone the deepcad repo and in the utils directory, run:

```
python export2step.py --src ../../CADBench/data/cad_vec_dedup_test --num -1
```

Note that you will need to modifiy line 37 in export2step.py to say "vec" rather than "out_vec". This will generate 6870 step files in data/cad_vec_dedup_test_step. 

Then run:

```
python get_bench/get_bench0_preprocess.py --deepcad_steps_dir data/cad_vec_dedup_test_step
```

This should save 6210 step files to data/cad_vec_dedup_test_step_cleaned. 

Then, run the following to generate the 8 isometric images for of the step files:

```
xvfb-run python scripts/generate_images_occ.py --input_parts data/cad_vec_dedup_test_step_cleaned --output_images data/bench0_occ_images --num_workers 32
```

Then run:

```
python get_bench/get_step_based_dataset.py --input_step_dir data/cad_vec_dedup_test_step_cleaned --image_dir data/bench0_occ_images --output_dir data/bench0
```

This will save the benchmark into data/bench0. To upload the selected steps to HuggingFace, run:

```
python scripts/upload_step_data_to_hf.py --dataset_dir data/bench0/steps --split_name bench0 --repo_name DeCoDELab/CADBench
```

### Bench 1: ABC

To get bench 1, perform the following steps - 

**1)** Download the STEP Files as well as OFS Files present in the ABC dataset from [this link](https://archive.nyu.edu/handle/2451/43778) Make sure all the STEP Files are in one folder, say `all_steps`, without subfolders. Let the file path to `all_steps` be `steps_dir`. The OFS Files should be left as they are downloaded from the ABC Website . Let the file path to the OFS Files be `yml_dir`.
   
**2)** Run the scripts for Mesh and Point Cloud Generation using the commands below. After running the command, the folders containing the mesh and point clouds will be present at the `mesh_dir` and `pc_dir`, which can be what you wish to be. Set the number of workers `num_workers` to a suitable number, say `N`.

```
python scripts/mesh_gen.py --input_dir steps_dir --output_dir mesh_dir --num_workers N
```

```
python scripts/pc_gen.py --input_dir steps_dir --output_dir pc_dir --num_workers N
```

**3)** Run the script `get_bench1_preprocess.py` to get two folders `abc_sketch_extrude` and `abc_other` at the path `out_dir_path`. `abc_sketch_extrude` contains the STEP Files that pass the filtering criteria and are created using sketch and extrude operations, and `abc_other` contains the STEP Files that pass the filtering criteria and are not created exclusively using sketch and extrude operations. Use the `dry_run` argument, if you  do not want to create the folders, but just see the pre-processing in action. 

```
python get_bench/get_bench1_preprocess.py --step_dir step_dir --yml_dir yml_dir --pc_dir pc_dir --mesh_dir mesh_dir --output_dir out_dir_path --num_workers N
```

**4)** Run the following script to generate the 8 isometric views for each of the samples in `abc_sketch_extrude` and `abc_other`. Let the paths to the images of `abc_sketch_extrude` and `abc_other` be `out_dir_path/abc_sketch_extrude_images` and `out_dir_path/abc_other_images`.

```
xvfb-run python scripts/generate_images_occ.py --input_parts abc_sketch_extrude --output_images `out_dir_path/abc_sketch_extrude_images`  --num_workers N
```

```
xvfb-run python scripts/generate_images_occ.py --input_parts abc_other --output_images `out_dir_path/abc_other_images`  --num_workers N
```

**5)** Finally, run the following code. Use the `out_dir_path` from Step 3.

```
python get_bench/get_bench1.py --out_dir_path out_dir_path --num_workers N --ops_cat abc_sketch_extrude
```

```
python get_bench/get_bench1.py --out_dir_path out_dir_path --num_workers N --ops_cat abc_other
```

### Bench 2: MCB
To get bench 2, first download the MCB A dataset from [this link](https://engineering.purdue.edu/cdesign/wp/a-large-scale-annotated-mechanical-components-benchmark-for-classification-and-retrieval-tasks-with-deep-neural-networks/#:~:text=Dataset%20A%20(Aggregated%20data%20from%20TraceParts%2C%203DWarehouse%2C%20and%20GrabCAD)). Rename the unzipped folder to `MCB_A` and move it under the data dir.

You should find 11700 mesh files in `data/MCB_A/test`. Next run watertight check and repair, checking for single bodies, and removal of duplicates using the following:

```
python get_bench/get_bench2_preprocess.py --mesh_dir data/MCB_A/test
```

This should leave 7106 .obj files in data/MCB_A/test_dedup. 

Then run the following script to generate the 8 isometric views for each of the samples:

```
xvfb-run python scripts/generate_images_occ.py --input_parts data/MCB_A/test_dedup --output_images data/MCB_A/test_dedup_images --num_workers 32
```

Then run the following to get the benchmark split of 3000 samples:

```
python get_bench/get_mesh_based_dataset.py --input_mesh_dir data/MCB_A/test_dedup --image_dir data/MCB_A/test_dedup_images --output_dir data/bench2
```


### Bench 3: Objaverse

To get bench 3, perform the following steps -

**1)** Filter Objaverse UIDs by category, download them, and repair the meshes to watertight GLBs. Omit `--uids` to let the script auto-filter (the result is cached to `data/objaverse_raw/filtered_uids.json` so the annotation fetch only runs once). The six categories included are: furniture-home, science-technology, architecture, cars-vehicles, electronics-gadgets, and weapons-military.

```
python get_bench/get_bench3_preprocess.py --out data/objaverse_raw --workers 8
```

**2)** Deduplicate the repaired GLBs.

```
python scripts/deduplicate_glbs.py data/objaverse_raw --output_dir data/objaverse_dedup
```

**3)** Eliminate singlebody meshes using the following:

```
python scripts/glb_singlebody.py --input_dir data/objaverse_dedup --output_dir data/objaverse_dedup_singlebody
```

**4)** Generate the 8 isometric view images for each mesh.

```
xvfb-run python scripts/generate_images_occ.py --input_parts data/objaverse_dedup_singlebody --output_images data/objaverse_dedup_singlebody_images --num_workers 32
```

**5)** Finally, run the following code to get the benchmark split of 3000 samples:

```
python get_bench/get_mesh_based_dataset.py --input_mesh_dir data/objaverse_dedup_singlebody --image_dir data/objaverse_dedup_singlebody_images --output_dir data/bench3
```

This will save the benchmark into data/bench3.

### Uploading Benches to Huggingface

Use `scripts/upload_data_to_hf.py` to upload a benchmark directory to a HuggingFace dataset repo.

The script expects a directory whose immediate subdirectories are treated as labels. For example, `data/bench0` contains `easy/`, `medium/`, and `hard/` — these become the `label` column in the dataset. If the directory contains CAD files directly with no subdirectories, the label will be empty.

Each row in the uploaded dataset corresponds to one CAD model and contains:
- `file_id`: the filename stem
- `label`: the subdirectory name (e.g. `easy`, `medium`, `hard`)
- `step`, `stl`, `obj`: binary file contents for each format present (others are `null`)

Example usage:

```
python scripts/upload_data_to_hf.py \
    --dataset_dir data/bench0 \
    --split_name bench0 \
    --repo_name DeCoDELab/CADBench
```

If the split already exists in the repo, the script will prompt you to either overwrite (`o`) or append (`a`) to it.

## Generate Modalities

### Generating Images

To generate the image modality, run the following to download stls:

```
python scripts/download_from_hf.py --split bench3 --out data/bench3_stls --format stl
```

Then run the following to generate gray scale isometric images from the meshes:

```
python  generate_modalities/generate_images.py --data_path data/bench3_stls
```

To generate multiview images, add --multiview as a flag on the end of the above command.

The images get saved to data/images/mesh_images_{data_path_root}_stl. For example, if data_path is data/bench3_stls, the images get saved to data/images/mesh_images_bench3_stls.

Then run the following to reupload to huggingface with the new images:

```
python scripts/reupload_split_with_images_to_hf.py --repo_name DeCoDELab/CADBench --split bench3 --image_dir data/imagesmesh_images_bench3_stls
```
